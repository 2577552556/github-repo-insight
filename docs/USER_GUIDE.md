## 1. 项目介绍

### 1.1 什么是 GitHub Repository Insight？

GitHub Repository Insight 是一个 GitHub 仓库健康检查工具，通过 AI 深度分析评估仓库质量，帮助你了解仓库的开发现状、社区活跃度和潜在风险。

### 1.2 主要功能

- **仓库信息展示** - 查看仓库基本信息、语言分布、Star/Fork 数量
- **健康评分** - 7 维度评分体系，全面评估仓库健康状况
- **AI 深度解读** - 基于 DeepSeek AI 生成专业的优势/风险/建议分析
- **项目类型检测** - 自动识别仓库类型（个人项目、企业开源、AI 平台等）
- **分析历史** - 保存历史分析记录，随时回溯查看
- **进度追踪** - 实时查看分析进度

### 1.3 评分等级

| 等级 | 分数范围 | 说明 |
|------|----------|------|
| A | 90-100 | 优秀，仓库健康状况极佳 |
| B | 80-89 | 良好，仓库整体健康 |
| C | 70-79 | 一般，存在一些改进空间 |
| D | 60-69 | 较差，需要关注问题 |
| E | 40-59 | 很差，存在严重问题 |
| F | 0-39 | 极差，仓库维护状况堪忧 |

## 2. 快速开始

### 2.1 环境要求

- **前端**: Node.js 20+
- **后端**: Python 3.12+
- **网络**: 可访问 GitHub API 和 DeepSeek API

### 2.2 启动服务

#### 步骤 1：启动后端服务

```bash
cd github-repo-insight
bash scripts/start-backend.sh
```

#### 步骤 2：启动前端服务

```bash
bash scripts/start-frontend.sh
```

#### 步骤 3：访问应用

打开浏览器访问 http://localhost:3000

### 2.3 停止服务

```bash
# 停止所有服务
bash scripts/stop-all.sh

# 或分别停止
bash scripts/stop-backend.sh   # 停止后端
bash scripts/stop-frontend.sh  # 停止前端
```

## 3. 功能使用

### 3.1 分析仓库

#### 操作步骤

1. 在首页的 URL 输入框中输入 GitHub 仓库地址
2. 点击「分析仓库」按钮
3. 等待分析完成，查看分析结果

#### 支持的 URL 格式

```
https://github.com/owner/repo
```

#### 示例

```
https://github.com/microsoft/vscode
https://github.com/facebook/react
https://github.com/torvalds/linux
```

### 3.2 查看分析结果

分析结果包含以下内容：

#### 仓库信息

- 仓库名称和描述
- 所有者
- Star、Fork、Watcher 数量
- 编程语言分布
- 许可证类型
- Topics 标签

#### 健康评分

7 个维度的评分：

| 维度 | 满分 | 说明 |
|------|------|------|
| 流行度 | 25 | Stars/Forks/Watchers |
| 活跃度 | 25 | Commit 频率和更新趋势 |
| 社区 | 15 | Contributors 数量和外部贡献者比例 |
| Issue 治理 | 10 | Issue 响应时间和关闭率 |
| PR 治理 | 10 | PR 合并时间和合并率 |
| 工程化 | 10 | License、README、Topics 完整性 |
| 发布维护 | 5 | 发布节奏和维护风险 |

#### AI 深度解读

- **优势分析** - 仓库的亮点和优势
- **风险分析** - 潜在的问题和风险
- **改进建议** - 具体的改进方向

每条结论都包含数据来源、置信度和因果关系标注。

### 3.3 管理分析历史

#### 查看历史记录

左侧边栏显示历史分析记录，点击记录可查看详情。

#### 新建分析

点击侧边栏的「新建分析」按钮，可开始新的分析。

#### 删除历史记录

点击记录右侧的删除按钮，可删除单条历史记录。

### 3.4 重新分析

在查看历史记录时，点击「重新分析」按钮可使用最新数据重新分析该仓库。

## 4. 设置配置

### 4.1 打开设置面板

点击页面右上角的设置图标，打开设置抽屉。

### 4.2 配置 GitHub Token

**作用**: 提高 GitHub API 请求频率限制（从 60 次/小时提升到 5000 次/小时）

**获取方式**:

1. 访问 https://github.com/settings/tokens/new
2. 设置 Token 名称
3. 选择 `public_repo` scope
4. 生成 Token
5. 在设置面板中粘贴 Token

### 4.3 配置 DeepSeek API Key

**作用**: 启用 AI 深度解读功能（可选，不配置仍可使用规则引擎评分）

**获取方式**:

1. 访问 https://platform.deepseek.com/api_keys
2. 创建 API Key
3. 在设置面板中粘贴 Key

### 4.4 凭据安全

- 凭据使用 AES-256-CBC 加密存储在本地数据库
- 主密钥自动生成并存储在 `backend/data/.credential_key`
- 凭据不会在网络上传输或显示

## 5. 常见问题

### Q: 分析失败怎么办？

**A**: 检查以下内容：

1. 仓库 URL 是否正确
2. 网络连接是否正常
3. GitHub Token 是否有效（如果配置了）
4. 查看后端日志 `logs/backend.log`

### Q: 为什么有些仓库评分较低？

**A**: 评分基于多个维度计算，可能原因包括：

- 仓库不活跃（很久没有更新）
- Issue 或 PR 响应时间较长
- Contributors 数量较少
- 缺少完整的文档（README、License 等）

### Q: AI 解读功能无法使用？

**A**: 确保已正确配置 DeepSeek API Key，且 API Key 有足够的调用额度。

### Q: 如何删除历史记录？

**A**: 在历史记录列表中，点击对应记录右侧的删除按钮。

### Q: 分析进度卡住了怎么办？

**A**: 刷新页面后重新发起分析。如果问题持续，检查后端服务是否正常运行。

### Q: 如何查看详细日志？

**A**: 日志文件位于：

- 后端日志: `logs/backend.log`
- 前端日志: `logs/frontend.log`

## 附录：评分指标说明

### 流行度维度 (25分)

- Stars: 0-15 分
- Forks: 0-5 分
- Watchers: 0-5 分

### 活跃度维度 (25分)

- 更新时间: 0-10 分
- 30 天 Commits: 0-15 分
- 90 天趋势: 0-5 分

### 社区维度 (15分)

- Contributors 数量: 0-10 分
- Issue 关闭率: 0-5 分

### Issue 治理维度 (10分)

- 响应时间中位数: 0-5 分
- 90 天关闭率: 0-5 分

### PR 治理维度 (10分)

- PR 合并时间中位数: 0-5 分
- 90 天合并率: 0-5 分

### 工程化维度 (10分)

- License: 0-4 分
- Topics: 0-3 分
- Description: 0-3 分

### 发布维护维度 (5分)

- 发布频率: 0-2 分
- 最近发布: 0-2 分
- 维护风险: -2-0 分
